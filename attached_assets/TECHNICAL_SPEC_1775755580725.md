# Golf Pool Scoreboard -- Technical Specification

This document is a complete technical spec for building a golf pool scoring web app. It is intended to be handed to an AI code generation tool (Bolt.new, Lovable, Replit, etc.) to produce a deployable application.

---

## 1. Tech Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| Framework | **Next.js 14+ (App Router)** | Full-stack React framework, API routes, SSR, deploys free on Vercel |
| Database | **Supabase (PostgreSQL)** | Free hosted DB, REST API, realtime subscriptions, dashboard for manual data viewing |
| Hosting | **Vercel (Hobby/Free)** | Zero-config Next.js deploy, serverless API routes, free `.vercel.app` domain |
| Styling | **Tailwind CSS** | Mobile-first utility classes, fast to build responsive layouts |
| Language | **TypeScript** | Type safety for scoring logic, API responses |

## 2. Database Schema (Supabase / PostgreSQL)

### Table: `tournaments`

```sql
CREATE TABLE tournaments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,                    -- e.g. "The Masters"
  year INTEGER NOT NULL,                 -- e.g. 2026
  espn_event_id TEXT,                    -- ESPN event ID for API calls
  status TEXT NOT NULL DEFAULT 'upcoming', -- 'upcoming' | 'active' | 'completed'
  current_round INTEGER DEFAULT 0,       -- 0=not started, 1-4=active round
  is_active BOOLEAN DEFAULT FALSE,       -- only one tournament active at a time
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Table: `pool_members`

```sql
CREATE TABLE pool_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,                    -- e.g. "Mullin"
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Table: `golfers`

```sql
CREATE TABLE golfers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  espn_id TEXT UNIQUE,                   -- ESPN athlete ID (e.g. "3470")
  name TEXT NOT NULL,                    -- display name from ESPN
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Table: `team_picks`

```sql
CREATE TABLE team_picks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tournament_id UUID REFERENCES tournaments(id) ON DELETE CASCADE,
  pool_member_id UUID REFERENCES pool_members(id) ON DELETE CASCADE,
  golfer_id UUID REFERENCES golfers(id) ON DELETE CASCADE,
  UNIQUE(tournament_id, pool_member_id, golfer_id)
);
```

### Table: `golfer_scores`

Stores cached scores from the ESPN API. One row per golfer per tournament per round.

```sql
CREATE TABLE golfer_scores (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tournament_id UUID REFERENCES tournaments(id) ON DELETE CASCADE,
  golfer_id UUID REFERENCES golfers(id) ON DELETE CASCADE,
  round_number INTEGER NOT NULL,         -- 1, 2, 3, or 4
  score_to_par INTEGER,                  -- null if not started, integer for to-par score
  holes_completed INTEGER DEFAULT 0,     -- 0-18
  is_cut BOOLEAN DEFAULT FALSE,          -- true if player missed the cut
  is_wd BOOLEAN DEFAULT FALSE,           -- true if withdrawn
  is_dq BOOLEAN DEFAULT FALSE,           -- true if disqualified
  tee_time TEXT,                         -- optional tee time string
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(tournament_id, golfer_id, round_number)
);
```

### Table: `round_metadata`

Stores per-round state, including the max score for cut player penalty calculation.

```sql
CREATE TABLE round_metadata (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tournament_id UUID REFERENCES tournaments(id) ON DELETE CASCADE,
  round_number INTEGER NOT NULL,         -- 1, 2, 3, or 4
  status TEXT DEFAULT 'not_started',     -- 'not_started' | 'in_progress' | 'completed'
  max_score_to_par INTEGER,              -- highest to-par score posted this round (for cut penalty)
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(tournament_id, round_number)
);
```

### Table: `manual_scores`

Separate table for manual mode. Stores team-level round scores entered by users.

```sql
CREATE TABLE manual_scores (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tournament_id UUID REFERENCES tournaments(id) ON DELETE CASCADE,
  pool_member_id UUID REFERENCES pool_members(id) ON DELETE CASCADE,
  round_1 INTEGER,                       -- to-par score, null if not entered
  round_2 INTEGER,
  round_3 INTEGER,
  round_4 INTEGER,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  updated_by TEXT,                        -- name of who last edited (optional, no auth)
  UNIQUE(tournament_id, pool_member_id)
);
```

### Table: `api_cache`

Tracks refresh state for the lazy refresh mechanism.

```sql
CREATE TABLE api_cache (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tournament_id UUID REFERENCES tournaments(id) ON DELETE CASCADE,
  last_fetched_at TIMESTAMPTZ,
  refresh_interval_minutes INTEGER DEFAULT 5, -- configurable interval
  UNIQUE(tournament_id)
);
```

## 3. ESPN API Integration

### Endpoint

```
GET https://site.api.espn.com/apis/site/v2/sports/golf/pga/scoreboard
```

Optional query param: `?dates=YYYYMMDD` to target a specific date.

No authentication required.

### Response Structure (relevant fields)

```typescript
interface ESPNResponse {
  events: Array<{
    id: string;                          // event ID
    name: string;                        // "Masters Tournament"
    status: {
      type: {
        id: string;                      // "1"=scheduled, "2"=in_progress, "3"=final
        state: string;                   // "pre", "in", "post"
        completed: boolean;
      }
    };
    competitions: Array<{
      competitors: Array<{
        id: string;                      // athlete ID
        athlete: {
          fullName: string;              // "Rory McIlroy"
          displayName: string;
          shortName: string;             // "R. McIlroy"
        };
        score: string;                   // overall to-par, e.g. "-11", "+3", "E"
        linescores: Array<{
          value: number;                 // raw stroke total for the round
          displayValue: string;          // to-par for the round, e.g. "-6", "+1", "E", "-"
          period: number;               // round number (1-4, or 5 for playoff)
          linescores: Array<{            // hole-by-hole scores
            value: number;
            displayValue: string;        // strokes on that hole
            period: number;              // hole number (1-18)
            scoreType: {
              displayValue: string;      // "+1", "-1", "E", etc.
            }
          }>;
          statistics?: {
            categories: Array<{
              stats: Array<{
                displayValue: string;    // last element is tee time string
              }>
            }>
          }
        }>;
      }>;
    }>;
  }>;
}
```

### Parsing Logic

```typescript
function parseESPNData(data: ESPNResponse, tournamentId: string) {
  const event = data.events[0];
  const competition = event.competitions[0];

  // Determine current round from event status
  // The highest period number with in-progress holes = current round

  for (const competitor of competition.competitors) {
    const espnId = competitor.id;
    const name = competitor.athlete.displayName;

    for (const linescore of competitor.linescores) {
      const roundNumber = linescore.period;
      if (roundNumber > 4) continue;   // skip playoff rounds

      const displayValue = linescore.displayValue;
      const holes = linescore.linescores || [];
      const holesCompleted = holes.filter(h => h.displayValue !== "-" && h.displayValue !== "").length;

      // Parse to-par score
      let scoreToPar: number | null = null;
      if (displayValue === "E") scoreToPar = 0;
      else if (displayValue === "-") scoreToPar = null;  // not started or cut placeholder
      else scoreToPar = parseInt(displayValue);           // handles "+3" and "-5"

      // Detect cut/WD status:
      // If this round's displayValue is "-" AND holesCompleted is 0
      // AND there is a completed prior round, player is cut
      const isCut = displayValue === "-" && holesCompleted === 0 && roundNumber > 2;

      // Extract tee time (last stat in statistics array, if present)
      let teeTime: string | null = null;
      const stats = linescore.statistics?.categories?.[0]?.stats;
      if (stats && stats.length > 0) {
        const lastStat = stats[stats.length - 1];
        if (lastStat.displayValue && lastStat.displayValue.includes(":")) {
          teeTime = lastStat.displayValue;
        }
      }

      // Upsert to golfer_scores table
      // { tournamentId, golferId, roundNumber, scoreToPar, holesCompleted, isCut, teeTime }
    }
  }

  // After processing all competitors, calculate max_score_to_par per round
  // for round_metadata table (used for cut player penalties)
}
```

### Lazy Refresh Mechanism

No cron jobs. The refresh is triggered by page loads with a cooldown:

```typescript
// API Route: GET /api/scores
async function getScores(tournamentId: string) {
  // 1. Read api_cache for this tournament
  const cache = await getApiCache(tournamentId);
  const now = new Date();
  const minutesSinceLastFetch = differenceInMinutes(now, cache.last_fetched_at);

  // 2. If enough time has passed, refresh from ESPN
  if (minutesSinceLastFetch >= cache.refresh_interval_minutes) {
    // Fetch fresh data from ESPN
    const espnData = await fetch(ESPN_SCOREBOARD_URL);
    await parseAndStoreESPNData(espnData, tournamentId);
    await updateApiCache(tournamentId, now);
  }

  // 3. Always return scores from database (cached data)
  return await calculateScoreboard(tournamentId);
}
```

**Refresh interval:** Default 5 minutes during active rounds. Configurable in the `api_cache` table. Set to a high value (e.g., 1440 minutes = once/day) on non-tournament days.

**"Last Updated" and "Next Update":** Returned with every scoreboard response:

```typescript
{
  lastUpdated: cache.last_fetched_at,           // "2026-04-09T14:30:00Z"
  nextUpdate: addMinutes(cache.last_fetched_at, cache.refresh_interval_minutes),
  scoreboard: [ ... ]
}
```

## 4. Scoring Engine

This is the core business logic. It runs server-side against cached database data.

### 4.1 Calculate Max Score Per Round (for cut penalties)

```typescript
function getMaxScoreForRound(tournamentId: string, roundNumber: number): number {
  // Query golfer_scores for this tournament + round
  // WHERE holes_completed = 18 (finished the round)
  //   AND is_cut = false AND is_wd = false AND is_dq = false
  // Return MAX(score_to_par)
  //
  // During a live round, also include in-progress scores:
  // WHERE holes_completed > 0
  //   AND is_cut = false
  // Return MAX(score_to_par)
  //
  // If no scores yet, return null
}
```

### 4.2 Calculate a Pool Member's Round Score (Best 4 of 6)

```typescript
function calculateMemberRoundScore(
  tournamentId: string,
  poolMemberId: string,
  roundNumber: number,
  maxScoreForRound: number | null
): { score: number | null; golferScores: GolferRoundDetail[] } {

  // 1. Get this member's 6 golfer picks for this tournament
  const picks = await getTeamPicks(tournamentId, poolMemberId);

  // 2. Get each golfer's score for this round
  const golferScores = [];
  for (const pick of picks) {
    const gs = await getGolferScore(tournamentId, pick.golfer_id, roundNumber);

    let effectiveScore: number | null = null;
    let isCutPenalty = false;

    if (gs && gs.score_to_par !== null) {
      // Golfer has a score for this round
      effectiveScore = gs.score_to_par;
    } else if ((gs?.is_cut || gs?.is_wd || gs?.is_dq) && maxScoreForRound !== null) {
      // Golfer is cut/WD/DQ -- assign max score penalty
      effectiveScore = maxScoreForRound;
      isCutPenalty = true;
    }
    // else: score remains null (round not started yet)

    golferScores.push({
      golfer_id: pick.golfer_id,
      golfer_name: pick.golfer_name,
      score_to_par: effectiveScore,
      holes_completed: gs?.holes_completed || 0,
      is_cut: gs?.is_cut || false,
      is_penalty: isCutPenalty,
      tee_time: gs?.tee_time || null,
    });
  }

  // 3. Sort scores ascending (lowest/best first), nulls go last
  const sortedScores = golferScores
    .filter(g => g.score_to_par !== null)
    .sort((a, b) => a.score_to_par! - b.score_to_par!);

  // 4. Take the best (lowest) 4 scores
  if (sortedScores.length < 4) {
    // Not enough scores yet to compute a full round score
    // Sum whatever we have (partial score during live play)
    const partialSum = sortedScores.reduce((sum, g) => sum + g.score_to_par!, 0);
    return { score: sortedScores.length > 0 ? partialSum : null, golferScores };
  }

  const best4 = sortedScores.slice(0, 4);
  const roundScore = best4.reduce((sum, g) => sum + g.score_to_par!, 0);

  // 5. Mark which golfers were counted vs dropped
  const countedIds = new Set(best4.map(g => g.golfer_id));
  for (const g of golferScores) {
    g.counted = countedIds.has(g.golfer_id);
  }

  return { score: roundScore, golferScores };
}
```

### 4.3 Calculate Full Scoreboard

```typescript
interface ScoreboardEntry {
  poolMember: { id: string; name: string };
  toPar: number | null;                   // total tournament score
  thru: string;                           // "F", "12", "-", or tee time
  today: number | null;                   // current round score
  rounds: Array<{
    roundNumber: number;
    score: number | null;
    golferDetails: GolferRoundDetail[];
  }>;
  rank: number;
}

async function calculateScoreboard(tournamentId: string): Promise<ScoreboardEntry[]> {
  const tournament = await getTournament(tournamentId);
  const members = await getPoolMembers();

  // Get max score for each round (for cut penalties)
  const maxScores: Record<number, number | null> = {};
  for (let r = 1; r <= 4; r++) {
    maxScores[r] = await getMaxScoreForRound(tournamentId, r);
  }

  const entries: ScoreboardEntry[] = [];

  for (const member of members) {
    const rounds = [];
    let totalToPar: number | null = null;
    let todayScore: number | null = null;

    for (let r = 1; r <= 4; r++) {
      const result = await calculateMemberRoundScore(
        tournamentId, member.id, r, maxScores[r]
      );
      rounds.push({
        roundNumber: r,
        score: result.score,
        golferDetails: result.golferScores,
      });

      // Accumulate total
      if (result.score !== null) {
        totalToPar = (totalToPar || 0) + result.score;
      }

      // "Today" = current active round score
      if (r === tournament.current_round) {
        todayScore = result.score;
      }
    }

    // Calculate "Thru" for this member
    const thru = calculateMemberThru(rounds, tournament.current_round);

    entries.push({
      poolMember: member,
      toPar: totalToPar,
      thru,
      today: todayScore,
      rounds,
      rank: 0,  // set after sorting
    });
  }

  // Sort by toPar ascending (lowest first), nulls last
  entries.sort((a, b) => {
    if (a.toPar === null && b.toPar === null) return 0;
    if (a.toPar === null) return 1;
    if (b.toPar === null) return -1;
    return a.toPar - b.toPar;
  });

  // Assign ranks (handle ties)
  let rank = 1;
  for (let i = 0; i < entries.length; i++) {
    if (i > 0 && entries[i].toPar !== entries[i - 1].toPar) {
      rank = i + 1;
    }
    entries[i].rank = rank;
  }

  return entries;
}
```

### 4.4 "Thru" Calculation

```typescript
function calculateMemberThru(
  rounds: Array<{ roundNumber: number; golferDetails: GolferRoundDetail[] }>,
  currentRound: number
): string {
  if (currentRound === 0) return "-";

  const currentRoundData = rounds.find(r => r.roundNumber === currentRound);
  if (!currentRoundData) return "-";

  const golfers = currentRoundData.golferDetails;

  // Collect thru states for each golfer
  const thruValues: Array<{ type: "finished" | "in_progress" | "not_started" | "cut"; value: number }> = [];

  for (const g of golfers) {
    if (g.is_cut) {
      thruValues.push({ type: "cut", value: 0 });
    } else if (g.holes_completed === 18) {
      thruValues.push({ type: "finished", value: 18 });
    } else if (g.holes_completed > 0) {
      thruValues.push({ type: "in_progress", value: g.holes_completed });
    } else {
      thruValues.push({ type: "not_started", value: 0 });
    }
  }

  const inProgress = thruValues.filter(t => t.type === "in_progress");
  const finished = thruValues.filter(t => t.type === "finished");
  const notStarted = thruValues.filter(t => t.type === "not_started");

  // All finished
  if (finished.length === golfers.length) return "F";
  // No golfer has in-progress or finished holes
  if (inProgress.length === 0 && finished.length === 0) return "-";
  // Some in progress: return minimum holes completed among in-progress golfers
  if (inProgress.length > 0) {
    const minHoles = Math.min(...inProgress.map(t => t.value));
    return minHoles.toString();
  }
  // Mix of finished and not started (no in-progress)
  return "-";
}
```

## 6. Frontend Components

### 6.1 Page Structure (`app/page.tsx`)

Single page application. No routing needed.

```
<TournamentHeader />
<RefreshStatus />
<ModeToggle />              // "Live" | "Manual" toggle
{mode === 'live' ? (
  <>

  </>

```

### 6.2 Component: `TournamentHeader`

- Tournament name in large text
- "Round X" indicator (or "Final" if completed)
- Dropdown to switch between tournaments (fetches from `/api/tournaments`)

### 6.3 Component: `RefreshStatus`

- "Last Updated: 2:30 PM ET" (relative or absolute, whichever is more readable)
- "Next Update: 2:35 PM ET"
- Subtle/muted styling, does not take up much space

### 6.4 Component: `ModeToggle`

- Simple toggle or segmented control: "Live" | "Manual"
- Defaults to "Live"
- Switching modes does not lose any data on either side

### 6.5 Component: `Leaderboard`

Mobile-first table. Columns:

| Rank | Name | To Par | Thru | Today | R1 | R2 | R3 | R4 |
|------|------|--------|------|-------|----|----|----|-----|

Formatting rules for all score cells:
- Negative: show as-is (e.g., "-6") in default text color
- Positive: prepend "+" (e.g., "+4") in default text color
- Zero: show "E"
- Null/empty: show "-" in a muted/lighter color (e.g., `text-gray-300`)

Clicking a row could scroll to that member's team card below.

### 6.6 Component: `TeamCard`

One per pool member, stacked vertically below the leaderboard.

**Header:** Member name (left), total tournament to-par score (right)

**Table:** Same columns as leaderboard but for individual golfers:

| Golfer | To Par | Thru | Today | R1 | R2 | R3 | R4 |
|--------|--------|------|-------|----|----|----|-----|

- Golfers sorted by To Par ascending (best first)
- The 2 dropped golfers per round: show their scores with a strikethrough or dimmed style
- Cut/WD/DQ golfers: show a small "CUT" / "WD" / "DQ" badge next to their name
- Penalty scores (assigned to cut players): show in a distinct style (e.g., italic or different color) so users know it is not a real score

**Collapsible:** On mobile, cards start collapsed showing just the header (name + score). Tap to expand and see the golfer detail table.

### 6.7 Component: `ManualLeaderboard`

Same layout as the live leaderboard, but:
- R1, R2, R3, R4 cells are editable input fields
- To Par (total) auto-calculates as the sum of the 4 round values
- Thru and Today columns are hidden (not relevant in manual mode)
- A "Save" button per row (or auto-save on blur)
- "Last edited by [name] at [time]" shown per row
- A text input for "Your name" so edits are attributed (no auth, just a name field)

### 6.8 Auto-Refresh (Frontend)

The frontend polls `/api/scoreboard` on an interval:
- While the page is visible and tournament is active: poll every 60 seconds
- The server-side lazy refresh decides whether to actually hit ESPN or serve cache
- Update the UI with fresh data on each poll
- Update "Last Updated" and "Next Update" timestamps

Use `visibilitychange` event to pause polling when the tab is not active.

## 8. Score Display Formatting (reusable utility)

```typescript
function formatScore(score: number | null): { text: string; className: string } {
  if (score === null || score === undefined) {
    return { text: "-", className: "text-gray-300" };  // muted placeholder
  }
  if (score === 0) {
    return { text: "E", className: "text-current" };
  }
  if (score > 0) {
    return { text: `+${score}`, className: "text-current" };
  }
  // score < 0
  return { text: `${score}`, className: "text-current" };  // negative sign included
}
```

Use this function for every score cell in the leaderboard, team cards, and manual mode.

## 11. Key Implementation Notes

1. **Supabase client:** Use `@supabase/supabase-js`. Create two clients: one with the anon key for public reads, one with the service role key for admin writes and ESPN data updates.

2. **ESPN API calls must happen server-side only** (in API routes), never from the browser. This avoids CORS issues and keeps the architecture clean.

3. **The `golfers` table is populated automatically** when a tournament is created. The admin fetches the field from ESPN, and all golfers are upserted by their ESPN ID. This solves the player name matching problem -- picks reference golfer records that came directly from ESPN.

4. **Supabase Row Level Security (RLS):** Enable RLS on all tables. Allow public SELECT on all tables. Allow INSERT/UPDATE only via the service role key (used by API routes). This prevents direct database manipulation from the browser while allowing public reads.

5. **Error handling:** If the ESPN API is unreachable, the lazy refresh should fail gracefully and serve stale cached data. Log the error but do not break the scoreboard.

6. **Time zones:** Store all timestamps in UTC. Display in the user's local time zone on the frontend using `Intl.DateTimeFormat` or a simple helper.
